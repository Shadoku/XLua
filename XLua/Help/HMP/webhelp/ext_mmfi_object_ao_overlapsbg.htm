Extensions Object Overlaps Background
 
XLua » Extensions » Object API » Active Objects

boolean object.overlapsBG ( [number xOffset [, number yOffset] ] )

Tests if the current sprite of the given object overlaps part of the background collision mask. The object must be on the same layer as the background to test positive.

An optional first and second parameter can perform the overlap test as if the object had been offset in the X and Y directions.

Parameters

xOffset An optional value that is added to the object's X coordinate prior to the overlap test. The object's coordinate is not actually changed.
yOffset An optional value that is added to the object's Y coordinate prior to the overlap test. The object's coordinate is not actually changed.

Return Values

boolean true if object is overlapping and obsticle, false otherwise 

Example

-- Assume we have a table of objects indexed by ID
object = mmf.objects[1]
 
-- Move an object down a pixel at a time, until we
-- hit an obsticle, or have moved 16 pixels.  We use
-- the optional yOffset parameter so we can test one
-- step ahead, and not actually embed into the obsticle.
for i = 1, 16 do
  if (object.overlapsBG(0, 1)) then
    break
  end
  object.y = object.y + 1
end
